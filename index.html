Types of errors

Syntactical

Been learning PHP Java Python or Rust?

Syntax errors are when you use symbols that aren't part of the language you are using.

Example: using x instead of * to multiple numbers

Additionally, when you finally figure out the correct symbols to use, but you're using them in the wrong way.

Example: if(num1 = num2)

Here we are assigning the value of num1, instead of comparing it to num2 with ==

Logic

So you've got the right symbols, you're using them correctly, so there isn't an actual error.

But now you are using them in a way that doesn't do the thing you're trying to accomplish.

Example: comparing the wrong variables for your conditionals, using a for loop but the condition is checking the wrong thing.

Execution

You've now gone off the reserve, and your code is just straight wrong. Maybe you started writing a function but forgot the closing }. Maybe you made a new const but didn't give it a value.

Ways we can debug code:

Check you code is actually running at all

Put a console.log() at the top of the file to make certain the file is running even a little bit!

console.log() 2: Electric Boogaloo

- One of the most common causes of the code not working the way you expect, is your code isn't being reached.

- You can put console logs inside functions, for loops and if statements that already exist in your code, to see if/when/how many times its being called

Example:

An if statement inside a for loop inside a function not doing what we want. There are 2 ways to fix a bug like this: work your way UP the hierarchy or work your way DOWN the hierarchy

Check the console in your browser

- If there is a literal error on your page, its going to tell you EXACTLY where the issue is.

- An error on line 10 will mean the code on line 20 WON'T RUN

- So its easy to mistake the lack of something happening with it being broken, instead of it being an entirely separate issue

Example:

Error on one line 22 breaking something later
